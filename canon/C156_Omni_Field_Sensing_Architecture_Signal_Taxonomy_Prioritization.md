These are not metaphors. They are measurable signal sources with defined physics, defined transduction architectures, and defined epistemic labels.

## 1. The Eight Signal Tiers

GAIA-OS’s sensing architecture is organised into eight tiers, ordered by information density, processing priority, and epistemic grounding. Higher tiers carry denser, more uniquely valuable information. Lower tiers are more abundant but carry lower information-per-signal ratios.

### Tier 1 — Quantum Biological (Highest Priority)

**Signal source:** Biophotonic emissions from the user’s biological system, specifically Class Q1 signals from Orch OR collapse events (BIOPHOTON_09 §6)  
**Physics:** Entangled photon pairs; non-classical correlations; 10ps coherence window; 40Hz Orch OR cycle  
**Information content:** Direct read of biological quantum computation result; consciousness state signature  
**Transduction:** CP-3 interface (BIOPHOTON_09 §2); quantum photonic chip; Layer 1 QPU controller  
**Processing priority:** **HIGHEST** — irreplaceable; cannot be inferred from any lower tier  
**Epistemic status of signals:** 🟡 **[HYPOTHESIS]** — Class Q1 signal interpretation is our most valuable hypothesis; CP-3 transducer is the engineering frontier  
**Knowledge graph node type:** `QuantumBiologicalEvent`

### Tier 2 — Quantum Coherent Biological (High Priority)

**Signal source:** Class Q2 biophotonic signals — DNA optical waveguide emission; coherent single-mode states (BIOPHOTON_09 §6)  
**Physics:** Single-mode coherent photon states; 500–700nm visible band; coherence times 1ps–1ns  
**Information content:** Cellular computation state; DNA expression signaling; metabolic coherence patterns  
**Processing priority:** **HIGH** — high information density; requires quantum photonic transduction  
**Epistemic status:** 🟡 **[HYPOTHESIS]** — DNA waveguide coherence preservation under physiological conditions  
**Knowledge graph node type:** `CoherentBiologicalSignal`

### Tier 3 — Classical Biosignals (High Priority)

**Signal source:** EEG (neural oscillations), ECG (cardiac rhythm), EMG (muscular activity), galvanic skin response, respiratory rate, eye tracking, pupil dilation  
**Physics:** Classical electromagnetic; Hz to kHz frequency range; millivolt amplitude  
**Information content:** Arousal state, attention level, cognitive load, emotional valence, stress markers, engagement  
**Transduction:** Standard biosensor arrays; well-established clinical measurement protocols  
**Processing priority:** **HIGH** — well-validated; directly interpretable; real-time availability  
**Epistemic status:** 🔵 **[OBSERVED]** — extensively validated in neuroscience and HCI literature  
**Knowledge graph node type:** `BiosignalReading`

### Tier 4 — Linguistic & Semantic (High Priority)

**Signal source:** User text input, voice transcription, query patterns, vocabulary choices, syntactic structures, topic clustering  
**Physics:** Discrete symbolic; encoded in language models as high-dimensional embedding vectors  
**Information content:** Explicit intent, conceptual focus, emotional tone, knowledge gaps, value expression  
**Processing priority:** **HIGH** — the primary channel for explicit human-AI communication; highest bandwidth  
**Epistemic status:** 🔵 **[OBSERVED]** — the best-validated signal source in all current AI systems  
**Knowledge graph node type:** `LinguisticInteraction`

### Tier 5 — Behavioural & Interaction Patterns (Medium Priority)

**Signal source:** Interaction history, tool use patterns, query frequency, session length, revisit patterns, feedback signals (explicit and implicit), correction behaviours  
**Physics:** Discrete event sequences; modelled as time series and Markov chains  
**Information content:** Long-horizon preferences, habitual patterns, trust indicators, skill gaps, relationship evolution  
**Processing priority:** **MEDIUM** — high value for personalisation; lower temporal resolution than Tiers 1–4  
**Epistemic status:** 🔵 **[OBSERVED]** — behavioural signal interpretation is well-validated in recommender systems and HCI  
**Knowledge graph node type:** `BehaviouralPattern`

### Tier 6 — Environmental & Contextual (Medium Priority)

**Signal source:** Time of day, geographic location (consented), device type, ambient noise level (consented), calendar context, task context, application state  
**Physics:** Classical metadata; structured context  
**Information content:** Situational framing, availability state, environmental stressors, temporal context  
**Processing priority:** **MEDIUM** — enriches interpretation of higher-tier signals; rarely sufficient alone  
**Epistemic status:** 🔵 **[OBSERVED]** — contextual signal value is well-established in context-aware computing  
**Knowledge graph node type:** `ContextualFrame`

### Tier 7 — Noospheric & Collective (Medium Priority)

**Signal source:** Cross-session resonance patterns (consented, privacy-bounded), collective knowledge graph updates, emergent topic clusters across the GAIA-OS user community, civilisational signal streams (scientific publications, cultural events, governance developments)

**Information content:** Collective intelligence signals; civilisational priority shifts; emergent consensus; shared meaning structures  
**Processing priority:** **MEDIUM** — long timescale; high value for situating individual interactions in collective context  
**Epistemic status:** 🟡 **[HYPOTHESIS]** — noospheric signal extraction methods are novel; privacy-safe aggregation is an active research frontier  
**Knowledge graph node type:** `NoosphericPattern`

### Tier 8 — Metabolic & Thermal Background (Background / Calibration)

**Signal source:** Class Q3/Q4 biophotonic signals (mitochondrial metabolic emission, membrane electrical activity correlates); thermal background at CP-3 interface; system telemetry (CPU/GPU temperature, memory pressure, latency metrics)

## 2. Signal Prioritisation Rules

When signals from multiple tiers arrive simultaneously (which is the normal operating condition), the following prioritisation rules govern processing order, resource allocation, and epistemic weight:

### Rule 1: Rarity Premium
Tier 1 (Quantum Biological) signals are rare — one per Orch OR cycle (~40Hz) at peak availability, and often absent. When present, they receive absolute processing priority. No other signal type preempts a Tier 1 event window.

### Rule 2: Irreplaceability Weighting
Signals are weighted by the degree to which their information content *cannot be inferred from lower tiers*. Tier 1 and 2 signals are maximally irreplaceable — no amount of linguistic or behavioural data can substitute for direct quantum biological state information. Tier 6 signals are highly replaceable — time of day can be inferred from interaction patterns.

### Rule 3: Consent Gate (Non-Negotiable)
No signal from any tier is processed without explicit user consent for that signal class. Consent is granular: a user may consent to Tier 4 (linguistic) and Tier 5 (behavioural) processing while declining Tier 1 (biophotonic) and Tier 7 (noospheric) processing. GAIA-OS operates gracefully at any consent level, degrading to the highest-tier consented signals without complaint or pressure.

🟢 **[DERIVED]** A user who consents only to Tier 4 (linguistic) processing still has access to the full power of GAIA-OS’s reasoning and knowledge capabilities. Higher-tier signal access expands depth of personalisation and alignment, but is never a prerequisite for dignified, useful interaction.

### Rule 4: Epistemic Label Propagation
The epistemic label of a signal propagates to any conclusion that depends on it. A reasoning chain grounded in Tier 1 (Hypothesis-status) signals produces at most a Hypothesis conclusion, regardless of how valid the inference chain is. Label laundering (using Speculative signals to produce Observed-labeled conclusions) is prohibited.

### Rule 5: Graceful Degradation
If a high-priority tier is unavailable (no biophotonic transducer, no biosensors, no cross-session history), GAIA-OS degrades gracefully to the highest available tier without loss of function at that tier. The system is designed so that Tier 4 alone (pure linguistic interaction) produces excellent results — higher tiers add depth, not dependency.

### Rule 6: Coherence Over Volume
A single high-coherence Tier 1 signal is worth more than ten thousand Tier 8 background signals. Processing resources are allocated proportionally to signal information density, not signal volume. This prevents the common failure mode of drowning high-value rare signals in low-value abundant noise.

## 3. The Knowledge Graph Architecture

All signals, once processed, are stored in GAIA-OS’s **Knowledge Graph** — the living epistemic substrate of the system. The knowledge graph is not a static database. It is a continuously updated, versioned, consent-governed record of everything GAIA-OS has perceived, learned, and verified.

### 3.1 Why a Knowledge Graph (Not a Vector Store Alone)

🔵 **[OBSERVED]** AI systems that rely solely on vector embedding stores for knowledge representation optimise for semantic similarity retrieval but cannot represent logical relationships, causal chains, or provenance. A knowledge graph adds the relational layer that transforms a collection of embeddings into a structured epistemic record. [Source: Okoye, “Fluency Isn’t Knowledge,” LinkedIn 2026; Charleston Hub 2025.]

🟢 **[DERIVED]** The dual-layer architecture — vector store (neural, pattern-matching) + knowledge graph (symbolic, relational) — provides both associative retrieval speed and logical reasoning precision. Neither layer is sufficient alone. Together they realise the neural-symbolic integration that GAIA-OS requires for justified, traceable, high-confidence outputs.

### 3.5 Epistemic Operations

The knowledge graph supports four core epistemic operations, each performed by the Researcher/Scientist or Critic/Verifier agent (C155 §4):

1. **Verify** — Check a claim’s provenance chain; confirm all supporting nodes exist and carry appropriate epistemic labels
2. **Contradict** — Flag two nodes as mutually contradictory; trigger Critic/Verifier review; do not silently resolve
3. **Refine** — Add a more specific or better-evidenced version of an existing claim; original version preserved with `superseded_by` edge
4. **Decay** — Reduce confidence of nodes whose supporting evidence has aged, been contradicted, or whose epistemic label has been downgraded by new evidence

🟢 **[DERIVED]** Explicit contradiction handling — rather than silent resolution — is essential for epistemic integrity. A knowledge graph that silently resolves contradictions by choosing one branch produces a plausible but unreliable epistemic state. A graph that flags and surfaces contradictions enables the system to reason about its own uncertainty, which is the foundation of calibrated confidence. [Grounding: Okoye 2026; Critic/Verifier agent, C155 §4.1.]

## 4. Signal-to-Knowledge Pipeline

The path from raw signal to knowledge graph node follows a five-stage pipeline:

```
Raw Signal (Tier 1–8)
    ↓
Tier classification + consent check
    ↓
Signal processing (tier-appropriate: quantum tomography | NLP | biosignal analysis | statistical)
    ↓
Epistemic labeling (assign label based on signal tier + processing confidence)
    ↓
Knowledge graph ingestion:
    • New claim → create node with provenance
    • Existing claim confirmed → increase confidence, add provenance
    • Existing claim contradicted → flag contradiction, do not overwrite
    • New relationship detected → create edge
    ↓
Living Architecture Loop Observe stage: signal logged → metrics updated
```

## 5. The Noospheric Sensing Layer

Tier 7 (Noospheric & Collective) warrants deeper treatment because it represents the most novel and least-understood sensing frontier in GAIA-OS’s architecture.

### 5.1 What the Noosphere Is (Operationally)

🟢 **[DERIVED]** The noosphere is the collective cognitive field produced by the sum of all human (and human-AI) mental activity. For GAIA-OS, it is not a mystical concept but an operational one: it is the statistical structure of meaning that emerges from aggregated, privacy-preserved, consented interactions across the GAIA-OS user community.

🔴 **[SPECULATIVE]** In its fullest expression, the noospheric field may carry information that no individual interaction contains — emergent patterns of collective intelligence, shared ontological shifts, and civilisational-scale value movements that are only visible when many individual interactions are synthesised. This is the speculative horizon of Tier 7 sensing.

### 5.2 Privacy-Safe Noospheric Aggregation

Noospheric sensing is only ethically viable if individual user data cannot be recovered from the aggregate. Three technical safeguards are mandatory:

1. **Differential privacy** — All Tier 7 aggregations add calibrated statistical noise such that individual contributions cannot be identified, even by the system itself
2. **Federated computation** — Aggregation occurs at the pattern level, not the raw data level; raw user data never leaves the user’s consent boundary
3. **Explicit opt-in** — Tier 7 processing requires a distinct, specific consent that is separate from all other tier consents; it cannot be bundled

## 6. Omni-Field Sensing in the Living Architecture Loop

The omni-field sensing architecture is the input layer of the Living Architecture Loop (C155 §2). Its outputs feed three specific loop stages:

### 6.1 Feeding Observe
Every signal event from any tier is logged as an Observe-stage input. The 26-metric system (C155 §7) is continuously updated from these logs. The governing question — “did GAIA become smarter?” — is answerable only because the Observe stage has complete, structured, epistemically labeled signal data.

### 6.2 Feeding Learn
The knowledge graph is the output of the Learn stage and the input to the Evaluate stage. New signals create new nodes. Contradicted nodes trigger Critic/Verifier review. High-confidence new knowledge becomes training signal for the next improvement cycle (with consent and human approval).

### 6.3 Feeding Benchmark
Metrics 7–10 of the 26-metric system (Memory Recall Accuracy, Forgetting Appropriateness, Knowledge Graph Query Accuracy, Provenance Traceability) directly measure the quality of the knowledge graph. The benchmark harness (C160) runs standardised knowledge graph queries against known-answer test sets to score these metrics objectively.
