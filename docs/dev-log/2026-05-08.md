## Gaps Identified

### Gap 1 — TaskScheduler Never Started ✅ FIXED
The `TaskScheduler` was instantiated inside every `GAIANRuntime.__init__()` but
its `run_forever()` async loop was never launched at boot or on lazy runtime
creation. Tasks submitted to the queue sat dormant forever.

### Gap 2 — Dual Memory Write Divergence ✅ FIXED
Every conversation turn wrote to two separate, non-communicating memory systems:
- `core/memory/memory_chroma.py` — ChromaDB, legacy, sentence-transformers
- `core/memory/store.py` — SQLite + sqlite-vec, Phase 3, full C17 audit

### Gap 3 — ActionGate Not Wired ✅ FIXED
`core/infra/action_gate.py` had never been instantiated, imported into the
request lifecycle, or connected to the Tauri frontend for RED-tier confirmation
dialogs.

## Final Architecture — Full Request Flow

```
Request → chat.py
  │
  ├─ _get_runtime(slug)              → GAIANRuntime (cached)
  │     └─ scheduler: run_forever()    → LIVE (Task 1)
  │     └─ rt.process(message)         → 18 engines
  │           └─ PolicyEngine            → soft gate (sets policy_flagged)
  │
  ├─ ActionGate.evaluate()           → Hard infrastructure firewall (Task 3)
  │     ├─ GREEN  → auto-approve, log, continue
  │     ├─ YELLOW → emit IPC → frontend dialog → wait 15s → approve on silence
  │     └─ RED    → emit IPC → frontend dialog → wait 60s → block on silence
  │           └─ if blocked → emit action_blocked SSE → stream exits
  │
  ├─ recall_for_prompt()             → MemoryBridge → MemoryStore (Task 2)
  │
  ├─ InferenceRequest(              → GAIAInferenceRouter
  │     gaian_system_prompt=           → T1 Canon
  │     result.system_prompt)          → T2 Criticality
  │                                    → T3 Noosphere
  │                                    → T4 Schumann/BCI
  │                                    → T5 Quintessence
  │                                    → LLM stream → token SSE events
  │
  └─ store_turn()                    → MemoryBridge → MemoryStore (Task 2)

ActionGate IPC round-trip (Tasks 3 + 4):
  gate.evaluate()
    → _emit_ipc("action-gate-confirm", {request_id, tier, ...})
        ├─ Production: _tauri_app_handle.emit()      → Rust → WebView
        └─ Dev:        [TAURI_IPC] log line          → app.ts log-bridge
    → window CustomEvent 'action-gate-confirm'
    → useActionGate hook → pending queue
    → ActionGateDialog renders (tier badge, description, countdown)
    → Human sovereign clicks Approve / Deny
    → resolve(request_id, approved)
        → POST /action-gate/respond
        → resolve_pending() → asyncio.Event.set()
        → _callback() unblocks → returns bool to ActionGate.evaluate()
        → gate_result propagates to chat.py → stream continues or exits
```
