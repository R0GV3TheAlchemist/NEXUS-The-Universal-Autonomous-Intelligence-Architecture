## Gaps Identified

### Gap 1 — TaskScheduler Never Started ✅ FIXED
The `TaskScheduler` was instantiated inside every `GAIANRuntime.__init__()` but
its `run_forever()` async loop was never launched at boot or on lazy runtime
creation. Tasks submitted to the queue sat dormant forever.

### Gap 2 — Dual Memory Write Divergence ✅ FIXED
Every conversation turn wrote to two separate, non-communicating memory systems:
- `core/memory/memory_chroma.py` — ChromaDB, legacy, sentence-transformers
- `core/memory/store.py` — SQLite + sqlite-vec, Phase 3, full C17 audit

### Gap 3 — ActionGate Not Wired ✅ FIXED
`core/infra/action_gate.py` had never been instantiated, imported into the
request lifecycle, or connected to the Tauri frontend for RED-tier confirmation
dialogs.

### Gap 4 — SovereignGuard + ActionGateDialog Never Mounted ✅ FIXED
Both `SovereignGuard` (Axiom I control bar) and `ActionGateDialog` existed as
components but were never rendered anywhere in the React tree. The React root
was `GaiaShell` — not `App.tsx` as initially assumed from reading only `src/`.

### Gap 5 — Native Tauri Emit Path Missing ✅ FIXED
The `action_gate_ipc.py` IPC layer had a log-bridge fallback but no real path
from Python to the Tauri WebView. A production build would silently fall back
to log parsing, which is dev-only. The Axum IPC bridge closes this completely.

## Final Architecture — Full IPC Chain

```
App boots
  ├─ Rust: start_ipc_bridge()          → Axum on 127.0.0.1:8009 — READY
  ├─ Rust: start_python_sidecar()      → Python FastAPI on :8008 boots
  ├─ Rust: polls /health until ready
  ├─ Rust: POST /internal/ipc-ready    → mark_ipc_bridge_ready() → flag = True
  └─ Rust: window.emit("sidecar:ready")

ActionGate fires (YELLOW/RED action)
  ├─ _emit_ipc()                       → POST 127.0.0.1:8009/emit
  ├─ Axum handler                      → app_handle.emit("action-gate-confirm", payload)
  ├─ WebView CustomEvent              → useActionGate hook → pending queue
  ├─ ActionGateDialog renders          → sovereign sees modal
  ├─ Approve / Deny                    → POST /action-gate/respond
  └─ asyncio.Event.set()              → gate resolves → stream continues or exits

Fallback path (dev / pre-bridge)
  └─ _ipc_bridge_ready = False         → [TAURI_IPC] WARNING log → log-bridge
```
