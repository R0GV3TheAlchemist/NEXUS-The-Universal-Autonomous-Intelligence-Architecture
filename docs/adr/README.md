# GAIA Architecture Decision Records (ADRs)

ADRs are the memory of why GAIA is designed the way it is.
Without them, every future contributor re-litigates decisions already made.

## How to Write an ADR

1. Copy `ADR-000-template.md`
2. Increment the number
3. Answer all four questions: **Problem, Decision, Alternatives, Consequences**
4. Set status to `Proposed` and open a PR
5. When merged: set status to `Accepted`
6. When superseded: update status and link to the new ADR — never delete

## Status Values

| Status | Meaning |
|---|---|
| Proposed | Under discussion |
| Accepted | Adopted — in effect |
| Superseded | Replaced by a newer ADR (link provided) |
| Deprecated | No longer relevant; not replaced |
